规则模板配置

💡 模板层设计

规则模板定义"怎么算"的逻辑，价格参数用变量占位（如 ${PRICE}）。
具体价格由客户定价层填充，实现同一API不同客户不同价。

接口组配置
主接口 计费接口（正常计费）
计费时按此接口调用次数计算
变量：${PRICE} 客户定价时填充具体数值
赠送接口配置（可配置多个）
赠送接口 1
自动等于主接口调用次数
变量：${OVERAGE_PRICE_1} 超出免费额度后按此价格计费
赠送接口 2
变量：${OVERAGE_PRICE_2}
┌─────────────────────────────────────────────────────────────┐ │ 模板编码：PRIMARY_BONUS │ │ 计费维度：API_GROUP（接口组） │ │ 组合策略：PRIMARY_BONUS（主接口+赠送接口） │ ├─────────────────────────────────────────────────────────────┤ │ 计费公式： │ │ │ │ 费用 = 主接口次数 × ${PRICE} │ │ + Σ max(0, 赠送接口调用 - 主接口次数) × 对应超量单价 │ │ │ │ 即： │ │ ① 主接口正常计费 │ │ ② 赠送接口免费额度 = 主接口调用次数 │ │ ③ 赠送接口超出免费额度的部分 × 超量单价 │ └─────────────────────────────────────────────────────────────┘
模板参数（客户定价时需填充）
参数变量 含义 示例值 说明
${PRICE} 主接口单价 0.10 主接口每调用一次的费用（元/次）
${OVERAGE_PRICE_1} 赠送接口1超量单价 0.20 赠送接口1超出免费额度后的费用
${OVERAGE_PRICE_2} 赠送接口2超量单价 0.15 赠送接口2超出免费额度后的费用
