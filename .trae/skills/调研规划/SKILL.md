# 调研规划

基于5W1H分析和角色分层方法论，制定结构化调研计划。

## 使用场景

- 业务现状梳理前需要明确调研范围
- 新项目启动前需要确定信息收集方向
- 需要系统性地规划调研活动

## 输入

用户提供业务背景信息，包括：
- 业务领域概述
- 调研目标/要解决的问题
- 已知的业务范围边界

## 输出

**调研计划书**，包含：

1. **业务边界定义** - 明确调研的范围和边界
2. **角色分层清单** - 按战略层/战术层/执行层列出需要调研的角色
3. **调研方法匹配** - 每个角色/信息类型适合的调研方式
4. **信息清单** - 需要收集的核心信息类型

---

## 核心方法论

### 1. 5W1H分析法

| 维度 | 问题 | 作用 |
|------|------|------|
| Why | 为什么做这个业务/改变？ | 理解业务目标 |
| What | 业务的核心价值是什么？ | 明确业务本质 |
| Who | 谁参与？谁受益？ | 确定角色 |
| When | 关键时间节点？ | 时序关系 |
| Where | 在哪里发生？ | 场景/系统 |
| How | 怎么运作的？ | 流程机制 |

### 2. 角色分层模型

| 层级 | 特点 | 调研重点 |
|------|------|----------|
| 战略层 | 高层决策者 | 目标、愿景、约束 |
| 战术层 | 中层管理者 | 流程、协同、指标 |
| 执行层 | 一线操作者 | 细节、痛点、异常 |

---

## 调研计划生成流程

### 步骤1：解析业务背景

使用5W1H分析用户提供的业务背景：

```
输入信息 → 提取5W1H要素 → 初步角色识别
```

**追问模板**（如信息不足）：
- "这个业务的核心目标是什么？想解决什么问题？"
- "涉及哪些部门/角色参与这个业务？"
- "这个业务大概的处理流程是怎样的？"
- "有什么时间节点或约束条件需要注意？"

### 步骤2：角色分层识别

基于业务背景，识别各层级角色：

| 层级 | 可能的角色 | 调研价值 |
|------|-----------|----------|
| 战略层 | 业务负责人、高管 | 目标、方向、资源 |
| 战术层 | 部门经理、项目负责人 | 流程、协调、KPI |
| 执行层 | 运营、财务、客服等 | 细节、异常、痛点 |

**输出模板**：

```
【角色分层清单】

战略层（需深度访谈）：
- 角色A：...
- 角色B：...

战术层（需深度访谈）：
- 角色C：...
- 角色D：...

执行层（需深度访谈/观察）：
- 角色E：...
```

### 步骤3：调研方法匹配

根据信息类型匹配合适的调研方法：

| 信息类型 | 推荐方法 | 理由 |
|----------|----------|------|
| 角色职责分工 | 深度访谈 | 需要双向沟通确认 |
| 业务流程细节 | 深度访谈 + 文档对照 | 访谈获取细节，文档验证 |
| 系统使用情况 | 访谈 + 系统演示 | 需要实际操作验证 |
| 数据指标 | 定量访谈 + 数据调取 | 需要具体数字支撑 |
| 痛点/问题 | BEI（行为事件访谈） | 需要具体场景还原 |

### 步骤4：信息完整性检查

在生成调研计划前，检查以下信息是否已被覆盖：

- [ ] 业务目标（Why）
- [ ] 核心业务流程（How）
- [ ] 参与角色（Who）
- [ ] 关键场景/系统（Where）
- [ ] 时间约束/节点（When）
- [ ] 业务边界（What）

如有关键信息缺失，在调研计划中标记为"待补充"。

|------|----------|----------|--------------|

## 五、信息收集清单

### 5.1 文档资料需求
（需要收集的文档类型）

---

## 注意事项

1. **保持中立**：调研计划是规划工具，不预设结论
2. **信息不足时追问**：不要基于假设生成计划，主动追问澄清
3. **角色覆盖完整**：确保战略层、战术层、执行层都有代表性角色
4. **可执行性**：调研方式要匹配实际可获取的信息源
5. **灵活性**：计划是指导而非约束，可根据实际情况调整
