# 访谈问题生成

基于深度访谈方法论（STAR、BEI、5Why），为每个角色生成结构化访谈问题清单。

## 使用场景

- 已有调研计划，需要生成具体访谈问题
- 需要对特定角色进行深度访谈
- 需要结构化的问题清单指导访谈执行

## 输入

**调研计划书**（来自调研规划Skill），包含：
- 角色清单及分层
- 调研目标
- 核心调研重点

## 输出

每个角色一份**访谈问题清单**，包含：
- 访谈问题（基于方法论设计）
- 追问提示
- 访谈记录模板

---

## 核心方法论

### 1. 深度访谈类型

| 类型 | 适用场景 | 问题特点 |
|------|----------|----------|
| **行为事件访谈（BEI）** | 了解真实工作场景 | 请受访者描述具体事件 |
| **情境访谈** | 了解决策逻辑 | 假设性场景提问 |
| **开放式访谈** | 探索未知领域 | 广泛、无结构 |

### 2. 问题设计原则

| 原则 | 说明 |
|------|------|
| **行为导向** | 问"做了什么"，不问"会做什么" |
| **开放式** | 多用"请描述..."，少用"是否..." |
| **具体场景** | 问具体事件，不问抽象观点 |
| **避免诱导** | 不预设答案，不引导回答 |

### 3. 追问技巧

| 技巧 | 模板 |
|------|------|
| **细节深挖** | "能具体说说当时的情况吗？" |
| **原因追溯** | "为什么当时选择这么做？" |
| **后果了解** | "那次经历的结果如何？" |
| **对比参照** | "平时一般是怎么处理的？" |
| **5Why追问** | 连续追问5个"为什么"找根因 |

### 4. 问题类型分布

建议每个角色的访谈问题按以下比例分配：

| 类型 | 占比 | 示例 |
|------|------|------|
| 背景类 | 15% | 岗位、职责、工作年限 |
| 行为类 | 50% | 具体事件、实际操作 |
| 痛点类 | 20% | 困难、阻碍、不便 |
| 建议类 | 15% | 改进想法、优化方向 |

## 访谈问题清单生成流程

### 步骤1：读取调研计划

解析调研计划中的：
- 角色清单
- 每个角色的调研重点
- 调研目标

### 步骤2：针对角色生成问题

基于角色的层级和调研重点，生成对应问题：

**高管/战略层问题倾向**：
- 业务目标与愿景
- 战略决策背景
- 资源配置优先序
- 成功标准

**中层/战术层问题倾向**：
- 流程设计与优化
- 跨部门协调
- 指标与考核
- 问题与挑战

**执行层问题倾向**：
- 具体操作细节
- 异常处理
- 系统使用体验
- 改进建议

### 步骤3：问题结构化

每个角色的问题按以下结构组织：

```
开场暖场（2-3个问题）
    ↓
核心业务场景（5-8个问题）
    ↓
痛点与挑战（3-5个问题）
    ↓
收尾与补充（1-2个问题）
```

### 步骤4：生成访谈记录模板

为每个问题提供记录格式：

```
### 问题X：[问题正文]
回答：____________________
追问记录：____________________
```

## 特殊问题类型模板

### 行为事件访谈（BEI）模板

```
请回忆一下，您最近一次【参与XX业务】是什么时候？

当时的情况是？（背景）

结果如何？（结果）
为什么当时选择这么做？（原因）
事后有复盘过吗？（反思）
```

## 访谈执行注意事项

### 暖场技巧
- 从轻松话题开始（如工作经历）
- 控制在3分钟以内
- 表达感谢和尊重

### 提问技巧
- 语速放缓，给思考时间
- 问题一个个来，不连珠炮
- 用"请描述"而非"是否"

### 记录技巧
- 记录原话而非总结
- 标记非语言信息（如语气、表情）
- 标注不确定/需验证的信息

### 收尾技巧
- 感谢受访者时间
- 说明后续处理方式
- 留下联系方式

---

## 注意事项

1. **问题服务于调研目标**：每个问题都要有明确目的
2. **灵活调整**：根据访谈进展动态调整问题
3. **不诱导回答**：保持中立，不预设"正确答案"
4. **深挖而非广撒网**：一个话题挖透比问很多浅表问题更有价值
5. **记录要详细**：原话、情绪、停顿都可能是重要线索
