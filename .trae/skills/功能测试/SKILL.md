# 功能测试

## 目的

解决"代码逻辑测试通过，但页面实际操作依然存在问题"的情况。通过完整的业务操作流程测试，验证功能在真实使用场景下是否可用。

## 触发条件

满足以下任一情况时，必须调用此skill：

1. 完成功能开发、bug修复、代码测试后
2. 用户反馈"明明说测试好了，但页面上操作有问题"
3. 任何涉及UI交互、页面操作的功能变更

## 测试流程

### 1. 确定测试目标

- 明确本次功能的核心业务操作路径
- 识别操作链路上的关键节点
- 列出可能影响操作成功的因素

### 2. 启动本地服务

```powershell
# 根据项目启动本地开发服务器
npm run dev
# 或其他适用的启动命令
```

### 3. 执行业务操作测试

按照真实用户操作路径逐步执行：

1. **进入功能页面** - 打开对应URL
2. **执行核心操作** - 点击、输入、提交等
3. **观察响应** - 页面反馈、数据变化、状态更新
4. **验证结果** - 确认是否符合预期

### 4. 记录问题

如果发现问题，必须：

- 记录具体的操作步骤
- 记录预期结果 vs 实际结果
- 定位问题代码位置
- 立即修复或报告

## 测试检查清单

- [ ] 页面正常加载，无JS错误
- [ ] 用户操作有正确反馈
- [ ] 数据正确保存/更新
- [ ] 页面状态正确切换
- [ ] 异常情况有合理提示

## 输出格式

测试完成后，必须报告：

```
【功能测试报告】
- 测试功能：<功能名称>
- 测试方式：<手动操作/自动化测试>
- 测试结果：✅ 通过 / ❌ 失败
- 问题列表：
  1. <问题描述> → <定位位置> → <修复方案>
```

## 注意事项

1. **不要仅依赖代码逻辑验证** - 必须实际执行页面操作
2. **测试用例要覆盖完整操作链** - 不能只测单个组件
3. **发现问题立即处理** - 不要遗留到下一轮
4. **必要时启动多个终端** - 保持服务运行的同时执行其他命令
