## 一、核心概念

### 1.0 账号 (accounts)
- **员工/个人**登录系统的账号
- 用户名：手机号（唯一）
- 一个账号可以绑定多个企业，也可以不绑定（纯个人账号）
- 账号可以使用的资金来源：
  - 未绑定企业：使用个人余额（**暂不支持，后续扩展**）
  - 绑定企业：使用所绑定企业的资金账户

### 1.1 客户 / 企业 (customers)
- 代表一个**企业客户**
- 一个企业下有多个部门，每个部门有多名员工
- 每个企业客户对应**一个资金账户**

### 1.2 企业资金账户 (customerAccounts)
- **企业层面**的财务账户，用于计费、结算
- 包含：余额、信用额度、支付类型（预付费/后付费）
- 一个企业客户对应一个资金账户

### 1.3 账号与企业绑定关系 (accountCustomers)
- 实现**账号与企业的多对多关系**
- 一个账号可以绑定多个企业，一个企业可以有多个账号
- 每个绑定关系有身份角色：admin（企业管理员）/ member（普通成员）

### 1.4 企业管理员
- 每个企业至少有一个管理员
- 企业管理员指定方式：
  1. 创建企业的账号自动成为该企业的管理员
  2. 企业申请更换 → 平台管理员审批后指定
  3. 平台管理员在后台直接创建企业并指定管理员
- 企业管理员可以审批账号绑定申请

### 1.5 产品 (products)
- 平台销售给客户的**API服务**
- 由平台进行定价和销售
- 一个产品可配置**多个通道**

### 1.6 通道 (channels)
- 供应商提供的**API接口**
- 通道是供应商的资源，平台采购后用于交付产品
- 一个通道可被多个产品使用（多对多）
- **业务层**：页面录入通道的API地址、入参出参等业务参数
- **技术层**：脚本负责实际调用，处理认证/加解密
- 脚本与通道**通过API URL协作**，脚本不需要"感知"新通道

### 1.7 供应商 (suppliers)
- 提供API通道的上游服务商
- 一个供应商对应一个脚本文件
- 密钥写在脚本中，由开发人员管理

### 1.8 脚本 (scripts)
- 处理供应商API的认证、加解密等技术细节
- 一个供应商对应一个脚本
- 脚本与通道通过API URL关联，不需要直接感知通道配置

## 二、业务流程

### 2.1 业务闭环
```
客户下单 ──→ 审核通过 ──→ 获得API Key ──→ 调用API
                                          ↓
                              运营给客户配置通道+路由策略
                                          ↓
                                    智能路由选择通道
                                          ↓
                              主通道优先，失败自动切换
                                          ↓
                                    调用明细记录
                                          ↓
                    客户账单(自动) ←─── 供应商账单(自动)
                          ↓                        ↓
                    客户付款                   付款给供应商
```

### 2.2 订单与扣费

| 支付类型 | 下单时 | 后续结算 |
|---------|-------|---------|
| 预付费 | 直接划扣账户余额 | 无需后续结算 |
| 后付费 | 不付款 | 按调用量从账户扣费 |

### 2.3 充值流程
```
管理员操作充值 → 生成充值记录(paymentOrders) → 增加账户余额
```

### 2.4 退款流程
```
管理员操作退款 → 生成退款记录(refundOrders) → 扣除账户余额
```

### 2.5 API调用流程
```
客户调用API ──→ 验证API Key ──→ 检查账户状态 ──→ 智能路由选通道
                                                        ↓
                                              按路由策略自动选择
                                                        ↓
                                              主通道优先，失败切换
                                                        ↓
                                                  调用通道API
                                                        ↓
                                              记录调用明细
                                                        ↓
                                              扣除账户余额(预付费)
```

## 三、支付与计费

### 3.1 支付类型（划扣时机）

| 类型 | 说明 |
|------|------|
| 预付费 (prepay) | 下订单时直接划扣账户余额/信用额度 |
| 后付费 (postpay) | 下订单时不付款，后续按调用量结算 |

### 3.2 计费规则（消耗方式）

**两个独立维度，相互配合**：

| 维度 | 概念 | 说明 |
|------|------|------|
| 预付费/后付费 | 划扣时机 | 下单即扣 vs 后续结算 |
| 按调用次数/包月包年 | 计费规则 | 决定如何消耗（如1元/次 或 固定月费） |

**示例**：预付费100元 + 按调用1元/次 = 下单划扣100元，限额100次调用

### 3.3 成本价格规则（与供应商结算）

| 规则类型 | 说明 | 示例 |
|---------|------|------|
| 按次计费 (per_call) | 每次调用固定费用 | 0.5元/次 |
| 按计费键 (per_key) | 按关键字收费，有封顶值 | 0.1元/车/次，日封顶0.3元 |
| 阶梯计费 (tiered) | 用量越大单价越低 | 1-1000次0.5元，1001-2000次0.3元 |
| 买赠规则 (buy_give) | 买量送量 | 买100次送10次 |

## 四、路由机制

### 4.1 通道池概念

- 产品对应多个通道，但这是**全量池**
- **运营按客户配置不同的通道子集**：不是所有客户都用全部通道
- 示例：产品有ABCD四个通道，客户1配置BCD，客户2配置AC

### 4.2 路由策略

- **运营给客户配置路由策略**（不是全局的，是客户级别的）
- 策略类型：价格优先（选成本低的）、性能优先等
- 客户只能调用为其配置的通道，并按策略自动选路

### 4.3 主备切换

- 主通道（isPrimary=true）优先使用
- 主通道失败时自动切换到备用通道
- 切换条件：响应码表示失败 **或** 响应时间超过阈值

### 4.4 智能路由模块

**尚未实现**，是后续开发的功能。

智能路由模块 = 路由策略的技术实现，根据客户配置的策略自动选择通道。

## 五、数据模型

| 数据表 | 作用 | 关联关系 |
|--------|------|----------|
| accounts | 账号（员工/个人登录账号） | 主表 |
| accountCustomers | 账号与企业绑定关系 | accountId → accounts, customerId → customers |
| customers | 企业客户基本信息 | 主表 |
| customerAccounts | 企业资金账户（余额、信用额度、支付类型） | customerId → customers |
| orders | 客户购买API的订单 | customerId → customers |
| apiKeys | API密钥（分企业API Key和个人API Key） | customerId → customers, accountId → accounts |
| products | 产品信息（名称、定价、状态） | 主表 |
| productChannels | 产品-通道关联表 | productId → products, channelId → channels |
| channels | 通道信息（供应商、URL、状态、响应码） | supplierId → suppliers |
| responseCodes | 响应码配置表 | channelId → channels |
| scripts | 脚本信息（关联供应商） | supplierId → suppliers |
| apiLogs / call_logs | 客户API调用明细 | customerId, productId, channelId |
| customerBills | 客户月度账单 | customerId → customers |
| customerBillItems | 客户账单明细 | billId → customerBills |
| supplierBills | 供应商月度账单 | supplierId → suppliers |
| billItems | 供应商账单明细 | billId → supplierBills |
| paymentOrders | 充值记录 | customerId → customers |
| refundOrders | 退款记录 | customerId → customers |

### 5.1 API Keys 分类

| 类型 | 关联 | 说明 |
|------|------|------|
| 企业API Key | customerId | 登录企业状态可用，调用时使用企业资金账户 |
| 个人API Key | accountId | 仅个人账号使用，调用时使用个人资金账户（暂不支持） |

### 5.2 账号绑定企业流程

| 申请类型 | 审批人 |
|---------|--------|
| 账号申请绑定企业 | 企业管理员 |
| 企业申请更换管理员 | 平台管理员 |

## 六、菜单结构

### 6.1 一级菜单
```
数据概览（固定第一位，默认展示，不可关闭）
▼ 供应商管理（可展开）
  ├── 供应商列表
  └── 通道列表（全局视图）
- 产品管理
- 订单审核
- 客户管理
```

### 6.2 客户管理
- 客户列表 - 查看所有企业客户及账户信息
- 客户账单 - 查看/生成客户月度账单
- 支付记录 - 查看充值记录
- 退款记录 - 查看退款记录

### 6.3 运营管理
- 供应商列表 - 供应商管理，含结算配置
- 通道列表 - 通道管理，含成本规则配置
- 产品管理 - 产品配置通道
- 订单审核 - 客户订单审核
- 调用明细 - API调用记录查询
- 供应商账单 - 供应商结算账单
- 客户账单 - 客户结算账单

### 6.4 交互规则
- 供应商管理为一级菜单，点击可展开/收起二级菜单
- 一级菜单选中时无背景高亮，只显示箭头旋转
- 二级菜单选中时有蓝色背景高亮
- 右上角蓝色圆形头像，点击下拉退出登录
