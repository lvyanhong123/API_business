# 账号体系审批流程改造设计

**日期**：2026-04-20
**版本**：v1.3.1

---

## 一、背景与问题

### 1.1 当前问题

| # | 问题 | 严重程度 |
|---|------|---------|
| 1 | 平台管理员审批了不该管的事（账号绑定申请应该由企业管理员审批） | 严重 |
| 2 | 强制绑定 vs 申请审批流程冲突（两个入口，两种效果） | 高 |
| 3 | 企业管理员没有审批入口（client.html 完全缺失） | 严重 |
| 4 | 审批状态没有同步反馈（申请者不知道进度） | 中 |
| 5 | 账号和绑定关系混在一个页面（操作混淆） | 中 |

### 1.2 改造目标

明确各角色职责：
- **企业管理员**：审批自己成员的绑定申请、申请更换管理员
- **平台管理员**：审批管理员变更申请、后台强制绑定
- **员工**：申请绑定企业、查看申请进度

---

## 二、审批流程设计

### 2.1 账号绑定申请流程

```
员工申请绑定
    ↓
binding.status = 'pending'
    ↓
企业管理员审批（client.html 审批Tab）
    ↓
通过 → binding.status = 'active'
拒绝 → binding 删除
```

### 2.2 管理员变更流程

```
企业管理员申请更换
    ↓
adminChangeRequest.status = 'pending'
    ↓
平台管理员审批（admin.html）
    ↓
通过 → 更新 accountCustomers 角色
拒绝 → adminChangeRequest 标记为 rejected
```

## 三、后端改造

### 3.1 新增数据表

**adminChangeRequests（管理员变更申请表）**

```javascript
{
  id: 1,
  customerId: 1,           // 企业ID
  currentAdminAccountId: 1, // 当前管理员账号ID
  newAdminAccountId: 2,    // 申请的新管理员账号ID
  status: 'pending',       // pending / approved / rejected
  createdAt: '2026-04-20',
  reviewedAt: null,
  reviewedBy: null
}
```

### 3.2 接口调整

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取待审批绑定 | GET | /accounts/pending-binds | 改为企业管理员查询本企业 |
| 审批绑定 | POST | /accounts/approve-bind/:bindingId | 改为企业管理员审批 |
| 拒绝绑定 | POST | /accounts/reject-bind/:bindingId | 改为企业管理员拒绝 |
| 申请更换管理员 | POST | /accounts/change-admin | 改为申请模式，创建待审批记录 |
| 获取变更申请列表 | GET | /admin/admin-change-requests | 新增，平台管理员用 |
| 审批变更 | POST | /admin/admin-change-requests/:id/approve | 新增 |
| 拒绝变更 | POST | /admin/admin-change-requests/:id/reject | 新增 |

### 3.3 权限校验调整

| 接口 | 校验逻辑 |
|------|----------|
| approve-bind | 必须是目标企业的 admin，且 binding 状态为 pending |
| reject-bind | 必须是目标企业的 admin |
| change-admin | 必须是目标企业的 admin |
| admin-change-requests/* | 必须是 platform admin |

## 四、前端改造 (client.html)

### 4.1 Tab结构调整

```
产品列表 | 我的订单 | API密钥 | 调用记录 | 企业管理
                                              ↓ 拆分为
                        企业管理（绑定企业列表）| 审批管理（仅管理员）
```

### 4.2 审批管理 Tab（新增，仅企业管理员）

- 入口：仅当用户是某个企业的 admin 时显示
- 功能：
  - 待审批绑定申请列表（成员申请加入本企业）
  - 通过/拒绝按钮
  - 操作后实时更新状态

### 4.3 企业管理 Tab 优化

- 申请绑定记录列表（显示申请进度：待审批/已通过/已拒绝）
- 申请更换管理员按钮（仅当前企业管理员可见）

### 4.4 申请更换管理员流程

1. 点击"申请更换管理员"
2. 弹窗选择目标成员（下拉列表，显示同企业已绑定成员）
3. 确认提交
4. 状态变为"等待平台管理员审批"
