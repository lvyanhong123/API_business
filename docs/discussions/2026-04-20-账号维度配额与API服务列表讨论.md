# 账号维度配额与API服务列表讨论

## 日期
2026-04-20

## 讨论主题
客户关联的账号和调用明细问题，以及API服务列表功能设计

## 业务梳理

### 核心概念
- **订单** = 购买记录（购买产品X多少次）
- **API服务/配额** = 产品的可用额度（订单的累加结果）
- 客户看"我的API服务"列表，展示已购买产品的剩余配额

### 账号维度
- 账号下订单获得的配额归属于该账号
- 账号用自己的API Key调用，消耗自己的配额
- 示例：账号X下订单1买产品A（100次），账号Y下订单2买产品A（50次）
  - X的API Key调用 → 消耗X的配额
  - Y的API Key调用 → 消耗Y的配额

### 配额消耗优先级
```
预付配额 → 用完
    ↓
检查预付自动续订
    ├── 有配置 + 资金账户余额充足 → 自动续订（预付） → 继续用预付配额
    └── 无配置 or 余额不足 → 跳过
        ↓
后付配额 → 用完
    ↓
检查后付自动续订
    ├── 有配置 + 资金账户（余额+信用额度）充足 → 自动续订（后付）
    │   └── 扣款顺序：优先扣余额，再扣信用额度
    └── 无配置 or 资金不足 → 拒绝调用
```

### 资金检查规则
- 预付自动续订：检查余额，扣余额
- 后付自动续订：检查余额+信用额度，优先扣余额再扣信用额度

### 预付与后付配额区别
- **预付费配额**：下单付款，剩余可结转
- **后付费配额**：信用配额，按实际调用量月底结算

### 月底账单结构（按账号维度）
```
企业账单（2026年4月）
├── 账号X
│   ├── 预付消费
│   │   ├── 产品A：购买100次，已用80次，剩20次，金额100元
│   │   └── 产品B：购买50次，已用50次，剩0次，金额50元
│   └── 后付消费
│       ├── 产品A：调用60次 × 单价 = 6元
│       └── 产品B：调用30次 × 单价 = 3元
│
└── 账号Y
    ├── 预付消费
    │   └── 产品A：购买80次，已用70次，剩10次，金额80元
    └── 后付消费
        └── 产品A：调用40次 × 单价 = 4元
```

## 技术方案

### 新增数据表
1. **accountProductQuotas** - 账号-产品配额表
   - accountId, productId, customerId
   - prepayRemaining (预付剩余)
   - postpayConsumed (后付已消耗)

2. **autoRenewalConfigs** - 自动续订配置表
   - accountId, productId, customerId
   - triggerType (ratio/count)
   - triggerValue
   - renewQuantity

### 改动点
- 订单新增accountId字段
- 审核订单时初始化账号配额
- API调用时按accountId区分消耗配额
- 账单按账号维度汇总
