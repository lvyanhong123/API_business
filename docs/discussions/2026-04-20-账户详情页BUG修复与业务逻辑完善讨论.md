# 2026-04-20 账户详情页BUG修复与业务逻辑完善讨论

## 问题描述

管理后台，账户详情页刷新的时候打不开，一片空白（在列表页点击可以打开该tab页）。

## 排查过程

### 第一次尝试：添加加载状态和错误处理

发现 `loadCustomerAccount` 函数在 API 调用失败时会弹出 alert，但没有渲染任何内容，导致页面空白。添加了：
- 加载状态提示（"加载中..."）
- 错误处理（"加载失败，请刷新重试"）

### 问题仍然存在

### 第二次排查：发现真正原因

浏览器控制台报错：
```
ReferenceError: Cannot access 'customerAccountId' before initialization
```

**根本原因**：`let customerAccountId = null;` 声明在 `loadCustomerAccount` 函数定义**之后**，JavaScript 的暂时性死区（Temporal Dead Zone）导致函数调用时无法访问该变量。

**解决方案**：将 `customerAccountId` 移到全局变量区域（第145行）。

## 业务逻辑讨论

### 1. 预付费/后付费字段位置问题

**讨论点**：为什么"预付费"会出现在资金账户里？它应该在订单级别。

**结论**：
- 资金账户 (`customerAccounts`) 只管：余额、信用额度、充值、退款
- 预付费/后付费是**订单属性**，由订单决定
- 已从 `customerAccounts` 相关代码中移除 `paymentType` 字段
- 客户级别的 `paymentType`（`customers.paymentType`）作为默认值保留，用于 API 调用时的业务判断

### 2. 账户状态字段问题

**发现**：`customerAccounts` 表实际上**没有 `status` 字段**，前端显示的"账户状态"读取的是不存在的字段。

**结论**：
- 账户状态不是资金账户的必要属性
- 冻结应该是客户级别的操作（`customers.status`）
- 已从前端删除账户状态显示

### 3. 废弃代码清理

**发现**：`showAccountModal` 和 `closeAccountModal` 函数已废弃（无调用入口），属于冗余代码。

**操作**：已删除废弃函数。

## 业务模型确认

```
客户 (customers)
├── 基本信息（公司名、营业执照等）
├── 状态 (status: active/frozen) ← 客户冻结
└── 支付偏好 (paymentType: prepay/postpay) ← 作为订单默认值

资金账户 (customerAccounts)
├── 余额 (balance)
└── 信用额度 (creditLimit)
    ← 不包含 paymentType（业务决定）
    ← 不包含 status（客户级别控制）

订单 (orders)
├── 支付类型 (paymentType: prepay/postpay) ← 必填
└── 其他订单信息...
```
