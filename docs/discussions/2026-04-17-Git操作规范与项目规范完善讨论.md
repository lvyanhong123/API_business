# Git操作规范与项目规范完善讨论

**日期**：2026-04-17
**主题**：Git操作流程规范化、skill命名、讨论记录机制

---

## 一、问题发现

用户发现我之前"拉取GitHub时没有先同步远程就查本地"，这是低级错误。

**问题链**：
1. 用户说"拉取 v1.2.7"
2. 我只查了本地 `git tag -l`
3. 没有先 `git fetch --tags origin`
4. 直接告诉用户"没有这个版本"

---

## 二、解决方案

### 2.1 新增 Pull/Fetch Workflow

在 `git-ops` skill 中添加拉取流程：

1. **Step 1**: 先 `git fetch --tags origin` + `git fetch origin` 同步远程
2. **Step 2**: 再检查 tag/branch 是否存在
3. **Step 3**: 然后 checkout
4. **Step 4**: 最后验证

### 2.2 Skill 重命名

`git-upload` → `git-ops`

理由：后续可能有更多 Git 操作，不仅上传和拉取

---

## 三、规则更新

### 3.1 新增 Git提交规则

**rules/project_rules.md** 新增：
```
### Git提交规则
- 用户说"提交GitHub"时，先记录讨论总结到 docs/discussions/，再执行提交
- 总结内容：本次讨论的决策、规则更新、新增/修改的skill内容
```

### 3.2 git-ops Skill 更新

**新增 Step 0**：在 Upload Workflow 的第一步（check status）之前，先记录讨论总结
