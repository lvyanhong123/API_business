# B端产品设计方法论

本文档记录《决胜B端》（杨堃著）的核心方法论，用于指导产品设计和需求分析。

---

## B端产品经理工作重点

```
分析并诊断业务问题 → 设计解决方案 → 执行并优化
```

> B端产品承担了帮助企业或机构解决某类经营管理问题的重任。

---

## B端产品设计流程

### 1. 业务调研

| 环节 | 说明 |
|------|------|
| 明确调研目标 | 梳理现有业务现状 / 总结业务问题 |
| 选取调研对象 | 战略层、战术层、执行层 |
| 确定调研方法 | 深度访谈、轮岗实习、调研问卷、数据分析、行业研究 |
| 执行调研计划 | 按计划执行调研 |
| 总结归纳输出 | 总结业务问题和痛点 |

### 2. 梳理业务现状

- 战略定位和目标
- 经营策略
- 管控模式
- 组织架构
- 业务流程
- 具体指标数据

### 3. 总结业务问题

找出业务现状与业务蓝图之间的差距，作为产品设计的出发点。

### 4. 核心流程（关键！）

- 使用**泳道图**绘制跨部门业务流程
- 从核心流程切入产品设计，不是从功能列表切入
- 只绘制关键场景和节点，不陷入细节流程
- 明确业务边界，才能定义产品边界

### 5. 产品定位

明确：
- 目标客户是谁？
- 解决了他的什么问题？
- 提供了什么价值？

### 6. 应用架构

技术层面的系统分层架构。

### 7. 功能模块设计

基于核心流程，划分功能模块。

### 8. 演进蓝图

| 期次 | 目标 |
|------|------|
| 一期 | 主流程闭环 |
| 二期 | 特殊流程完善 |
| 三期 | 优化体验 |

### 9. 数据建模（ER图）

业务实体和数据单元的对应关系。

### 10. 流程和角色设计

基于角色的页面流转图。

### 11. 权限设计

---

## 核心观点

1. **从核心流程切入，不是从功能列表切入**
2. **泳道图的作用**：通过总结的业务问题以及解决思路，以泳道图形成业务流程并优化流程
3. **产品定位是产品设计的指导思想和灵魂**
4. **业务边界定义产品边界**
